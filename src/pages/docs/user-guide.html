User Guide

FreeFSM is organized into modules accessible from the sidebar. This guide covers each module's purpose and key features.

Dashboard

The Dashboard is the first screen you see after logging in. It provides an overview of key metrics including KPI cards for revenue, job counts, and status breakdowns. The calendar shows upcoming schedules and the global search bar lets you find customers, jobs, and invoices from one place. Quick action buttons provide one-click access to common tasks.

Customers

The Customers module manages your customer database. Each customer record includes:

  • Contact information — Name, email, phone, company name
  • Addresses — Billing and service addresses
  • Status — Lead, opportunity, customer, lost, inactive
  • Pipeline tracking — Pipeline status and lead source
  • Contacts — Multiple contacts per customer
  • Locations — Polymorphic locations attached to any object
  • Custom fields — Extensible fields stored as JSONB

Search across all customers, filter by status, and navigate results with HTMX-powered pagination.

Jobs / Work Orders

Jobs are the core operational unit in FreeFSM. Each job represents a field service visit or work order.

  • Status workflow — Configurable statuses (not hardcoded enums)
  • Visits — Multi-visit jobs with arrival windows per visit
  • Assignments — Teams and individual technicians assigned via JSONB
  • Subtasks — Checklist items on jobs for step-by-step tracking
  • Scheduling — Start/end times, due dates, arrival windows
  • Billing type — Configurable billing model per job
  • Notes — Internal notes and field notes visible to technicians

Schedule

The Schedule displays jobs in calendar and list views. Click any job card to see details. Filter by team, user, or status to focus your view. Powered by Alpine.js for a smooth client-side experience.

Assets

Track customer equipment with detailed asset records. Each asset includes manufacturer, model, serial number, install date, warranty information, and service history. Assets can be categorized, linked to customers and jobs, and tracked throughout their lifecycle.

Projects

Group related jobs under a project for better organization. Projects link to a customer and show progress through a completion percentage. View all nested jobs and their statuses from a single project overview page.

Estimates

Create estimates using line items from your pricebook. Line items are snapshotted at creation time to preserve historical accuracy.

  • Line items editor — Add items from the pricebook with quantities and prices
  • Status workflow — Draft, sent, approved, rejected
  • Tax calculation — Per-item or overall tax rates
  • PDF generation — Generate professional PDF estimates
  • Conversion — Convert approved estimates to jobs and invoices

Invoices

Generate invoices from line items, with payment tracking and status management.

  • Line items — Copied from pricebook as snapshots
  • Payments — Record partial or full payments with method and reference
  • Status workflow — Draft, invoiced, paid, void
  • PDF generation — Basic PDF via maroto (MVP), Chromium-based planned

Items / Pricebook

Manage your service and product catalog. Each item includes:

  • Type — Service or product
  • SKU — Unique stock keeping unit identifier
  • Pricing — Unit price and cost tracking
  • Tax — Taxable flag and tax rate
  • Status — Active or inactive

Timesheets

Track technician time with clock in/out functionality. Timesheets capture start and end times, link to specific jobs, and optionally record GPS coordinates. Manual entry is supported for offline or retrospective time recording.

Tags

Color-coded tags can be applied to customers, jobs, assets, invoices, and any other object type. Tags are admin-configurable and make filtering and categorization visual and fast.

Custom Fields

User-defined fields extend any entity type with the data you need. Configure field definitions per object type in settings, then populate them on individual records.

Comments

Threaded comments on any object enable collaboration. Each comment shows authorship and timestamp for a complete audit trail.

User Management

Manage users with role-based access control. Three roles are supported:

  • Admin — Full system access
  • Tech — Field technician access
  • Dispatcher — Scheduling and dispatch access

Additional features include welcome emails, password policies, and force password change on first login.

Company Settings

Configure your company profile, branding (logo, colors), email server settings, timezone, document numbering schemes, and security policies. The dark mode theme toggle persists across sessions.

Activity / Audit Log

Every create, update, and delete operation is recorded in the audit log. Per-entity activity widgets appear on show pages for customers, jobs, invoices, and other entities, giving you the full history of any record. A dedicated admin-only activity page provides a system-wide audit trail, and recent activity panels appear on every list page for at-a-glance context.

File Attachments

Upload files to customers, jobs, estimates, invoices, and assets. Supported file types are governed by a MIME whitelist for security. Images and PDFs generate inline previews directly in the browser. Files are stored on disk under the directory specified by FREEFSM_UPLOAD_DIR, with a configurable maximum upload size via FREEFSM_MAX_UPLOAD_SIZE (default 25 MB).

Soft-Delete / Archive

Business entities are archived rather than permanently deleted. Archived records are hidden from normal list views but remain in the database. An admin can restore any archived entity from a banner displayed on its show page, making accidental data loss reversible.

Auth

FreeFSM uses bcrypt-hashed passwords and HTTP-only session cookies for authentication. The initial admin account is created via a setup token. CSRF protection is enabled on all state-changing requests.
