Tech Stack Rationale

Go

Single static binary with zero runtime dependencies. Cross-compiles to both FreeBSD and Linux from a single build machine. Excellent standard library with built-in HTTP server, templating, and testing.

chi Router

Lightweight, idiomatic, and composable. Middleware chaining and route grouping map cleanly to the module structure.

PostgreSQL + JSONB

Line items, payments, visits, assignments, and custom fields are stored as JSONB columns. These are value objects — always fetched with their parent and never queried independently. JSONB avoids joins, enforces snapshot semantics, and supports indexing when needed. This is the same approach used by Invoice Ninja in production.

ent ORM + entpoly

Type-safe code generation for database access. The entpoly extension adds first-class support for polymorphic relations (tags, comments, locations, custom fields). Eager loading with batching eliminates N+1 query problems.

Templ + HTMX + Alpine.js

Templ provides compile-time safety for HTML templates. HTMX enables server-driven interactivity without writing JavaScript — links and forms issue AJAX requests and swap HTML fragments. Alpine.js handles client-side state for the calendar and line items editor. Zero npm dependencies, no build step for front-end code.

Key Database Design Decisions

JSONB Columns (not separate tables)

Several entity attributes use JSONB instead of separate tables:

  • invoices.line_items — Item snapshots with quantities, prices, taxes
  • invoices.payments — Payment records attached to invoices
  • estimates.line_items — Same structure as invoices
  • jobs.visits — Multi-visit scheduling with arrival windows
  • jobs.assignments — Team and user assignments
  • entities.custom_fields — Extensible fields on customers, jobs, invoices

Polymorphic Relations

Tags, comments, and locations use a polymorphic pattern with object_type + object_id columns, enabling a single table to attach to any entity type.

Status Workflow System

Jobs, estimates, and invoices share a configurable workflow system. status_workflows define the available statuses per entity type, with ordering and color. Statuses can be system defaults or user-defined custom workflows.

Line Items Architecture

When an estimate or invoice is created, line items are copied from the pricebook as JSONB snapshots. This ensures that changing a pricebook price never retroactively alters existing documents. Each line item captures: item ID, title, description, unit price, quantity, taxable flag, tax rate, discounts, and surcharges.

Key Patterns

Polymorphic Relations

Built via entpoly. Tags, custom fields, comments, and locations use object_type + object_id to attach to multiple entity types.

Assignment Pattern

Jobs, material lists, and subtasks use JSONB arrays with team_id + assigned_members[] for flexible team-based assignments.

Standardized Query System

All list endpoints support: limit, page, search, sort_by/sort_dir, filter[], rel[] (eager loading), calculate_count.
