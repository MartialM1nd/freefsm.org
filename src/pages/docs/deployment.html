Deployment

FreeFSM compiles to a single static binary with embedded assets, making deployment straightforward on both Linux and FreeBSD.

Linux (systemd)

The Makefile includes an install-linux target that builds the binary and installs it alongside a systemd service unit:

make install-linux

This installs:

  • The binary to /usr/local/bin/freefsm
  • A systemd unit to /etc/systemd/system/freefsm.service

Start the service:

systemctl start freefsm
systemctl enable freefsm

FreeBSD (rc.d)

Similarly, the install-freebsd target handles FreeBSD deployments:

make install-freebsd

This installs:

  • The binary to /usr/local/bin/freefsm
  • An rc.d script to /usr/local/etc/rc.d/freefsm

Start the service:

service freefsm start
sysrc freefsm_enable=YES

Makefile Targets

Target Description
make buildGenerate ent, templ, then compile Go binary
make runBuild and run the server
make install-linuxBuild and install for Linux with systemd
make install-freebsdBuild and install for FreeBSD with rc.d
make fmtFormat Go source code
make lintRun go vet
make cleanRemove build artifacts

Environment File

Create a .env file in the same directory as the binary, or set environment variables via your service manager:

FREEFSM_DB_HOST=localhost
FREEFSM_DB_PORT=5432
FREEFSM_DB_NAME=freefsm
FREEFSM_DB_USER=freefsm
FREEFSM_DB_PASSWORD=your_password_here
FREEFSM_SESSION_SECRET=your_secret_here
FREEFSM_SETUP_TOKEN=your_setup_token_here

See the Configuration Reference for all available variables.
